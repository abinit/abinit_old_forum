Hi Ron,
Camilo Espejo, who implemented the vdW in Abinit, has a technical problem to reply in the forum, so please find here his response:
Hi (recohen) Ron,
It's been our fault not to stop Abinit execution if vdw_xc=1,2. Currently if any of these values is set, Abinit will not actually call for the vdW-DF module. This issue will be corrected asap in the development version .
Some bugs remain to be solved before we can make the implementation of vdW-DF publicly available, since the values of the vdW energy we are getting now are not fully reliable. I think this will take two more months. I'll let you know when the fully operational implementation can be found in the development version.
Regards,
Camilo.
