I personally don't know anything about TECPLOT, and I don't know if someone here knows about it. A quick search on the internet showed that it seems to accept a text file format as input (see http://local.wasp.uwa.edu.au/~pbourke/dataformats/tp/ for details).

I thus suggest you to do one of the following:
  • to use another visualization tool, like V_Sim;
  • to extend Cut3D to make it able to write TECPLOT files;
  • to write a converter between the _DEN format of Abinit and the TECPLOT format.

If you choose one of the two last options, it would be nice for everybody that your contributions be included officially into Abinit.
