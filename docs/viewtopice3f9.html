Hello Goodluck,

the old netcdf format has been disabled - what you should use is the new standard which is ETSF-io. Abinit can produce it directly (see accesswf variable and configure script options) so you don't even have to go through cut3d.

The format is much more complete, has a common set of structural data for potentials wf density etc... and can be read by several other codes.
