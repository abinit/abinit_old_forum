Dear Andy,

The error you see says that in the case of non-linear computations (optdriver=5) with the "FULL DFPT" method (usepead=0) the variable "occopt" should be 1 (0 should be fine too, I think).

Indeed this routine has been validated only in the case of insulators. It's true that in principle one can use metallic occupations with a very small smearing to describe an insulator, but the non-linear implementation has not been checked and validated in that case yet.

There is still some work to do in the non-linear implementation to make it available for more cases. We are working on it, and this specific problem will be taken into consideration. However it won't be released soon...

Best wishes,
Lucas Baguet
