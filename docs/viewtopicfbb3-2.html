Dear bahaar,

as indicated in the README file :

* parallel

- start as administrator , a "command prompt" to start smpd
bin\smpd -install

- as "user"
cd share\example
..\..\bin\mpiexec -localonly 2 ..\..\bin\abinit < "t_kpt+spin.files"


another solution is to install ( as admin) the mpich2 package for windows ( http://www.mcs.anl.gov/research/projects/mpich2/ )
the smpd will be started by the installer ( as windows service )

Hello,

First of all, I'm just trying the recipe with my PC with a fresh windows 7 installation and the "abinit-6.6.3_i686_cygwin_gnu4.5.zip" package
bahaar wrote:For parallel i have a question. I use the "bin\smpd -install" command as determined in readme.

except for the command "bin\smpd -install" when Windows firewall asks me the permission to open the access to the private network
everything works correctly

caution : this command must be run as administrator ( as mentioned in the Readme file )

Then i run this command: " D:\abinit-6.6.3_i686_cygwin_gnu4.5\abinit-6.6.3_i686_cygwin_gnu4.5\bin>mpiexec -localonly 2 D:\abinit-6.6.3_i686_cygwin_gnu4.5\abinit-6.6.3_i686_cygwin_gnu4.5\bin\abinit < t_kpt+spin.files.


there is another problem with your cmd : it seems that you are in the bin folder to run this command ! but the file "t_kpt+spin.files" is in the share\example folder !

I suggest to run abinit like that :

Code: Select all

cd D:\abinit-6.6.3_i686_cygwin_gnu4.5\abinit-6.6.3_i686_cygwin_gnu4.5
cd share\example
..\..\bin\mpiexec -localonly 2 ..\..\abinit < "t_kpt+spin.files"


regards
