For the screening, there are ways to split the calculation into independent jobs.
Look at the variables nqptdm and qptdm. It allows you to compute the screening for each q-point individually, then you can merging them at the end with mrgscr. This procedure is more safe for very large calculations.
Also, if a screening calculation crashes, it may have produced a _SUS file. It's the susceptibility, which has to be inversed to give the screening. You can restart the run from that point, using irdsuscep.
