Hi-

I have been struggling to get a parallel version of abinit compiled such that it can properly hand fft and band parallelization. I have had no problem getting it to parallelize over k-points, but activating npband seems to be a poison pill. I am trying to do defect supercell calculations for MgSiO3 perovskite, which has a large enough cell to require fft parallelization due to memory limitations (since we have 12 gigs of memory per 8 processor core). I am using abinit 6.6.3 and compiling with the updated fortran compiler 11.1.075 and using the mkl linear algebra libraries (openmpi was also compiled using the same version of ifort). The system admin and I have tried many different ways of compiling, and none have worked (older ifort, w/ or w/out mkl, etc...). Based on forum posts, our most recent try was configured with the following command:

../configure --enable-64bit-flags --enable-mpi --enable-mpi-io --with-linalg-flavor=mkl --with-linalg-incs="-I/share/apps/intel/Compiler/11.1/075/mkl/include" --with-linalg-libs="-L/share/apps/intel/Compiler/11.1/075/mkl/lib/em64t -lmkl_intel_lp64 -lmkl_blacs_lp64 -lmkl_intel_thread -lmkl_core -liomp5 -lpthread -lmkl_blas95_lp64 -lmkl_lapack" --with-mpi-prefix=/share/apps/openmpi-intel11.1.075/

This results in a successful configuration that also compiles. With that binary, I can run jobs with only kpt parallelization. But when I try a job that needs fft and band parallelization, the job hangs and exits within one or a few iterations. I have tried running the code with both ethernet and myrinet (the standard on our cluster), and this changes the error message but the job still fails.

Below I have copied the end of the failed job log files and the output of the config script. I have also attached my input file. I would greatly appreciate any help you can give me !!!

Best,
Aaron Wolf
