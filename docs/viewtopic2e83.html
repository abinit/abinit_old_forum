checking whether the C compiler supports MPI... no

configure: WARNING: MPI support is broken!


You already have a problem here since MPI is not usable. Usually it is because the path to the MPI librairies is not correctly defined. If you have openMPI for instance, you can specify in the configure command:

Code: Select all

../configure --enable-mpi --with-mpi-prefix=/path/to/mpi/executable


It should use mpif90 to compile.

You may find the path to the MPI executable by echoing $PATH.

Post by Boris » Wed Jul 27, 2011 9:35 pm

Then try to explicitely give the path to the compilers, for instance:

../configure --enable-mpi FC=/opt/OpenMPI/openmpi-1.4.3-intel/bin/mpif90 CC=/opt/OpenMPI/openmpi-1.4.3-intel/bin/mpicc CXX=/opt/OpenMPI/openmpi-1.4.3-intel/bin/mpic++


and changing the path accordingly. First check that mpif90, mpicc and mpic++ are available in your directory.

Also, if you're compiling on a cluster or something alike, you might need to load the open-mpi module.
