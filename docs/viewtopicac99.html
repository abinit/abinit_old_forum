errors are usually printed in the log file, are you saving the log file? Your execution line should be something like

abinit < foo.files &> log &

so that the file log gets all the error messages. Also, I suggest you not use such a huge number of energies in your convergence test. Try something like

ndtset 5
ecut: 40 ecut+ 5

Probably there is some inconsistency in your input file, but it is being printed to the log file (not the .out file). It is taking a minute to run because it is first doing some checks on all 40 of your input files. By reducing the number of input files it generates you can debug your problem much faster.
