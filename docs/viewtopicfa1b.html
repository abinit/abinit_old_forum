Dear Mutta

I just tried running your inputs after considerable changes were made. Good news is that the runs finished without errors.
I used the version 6.4.1 of abinit and the psp files were 82pb.pspnc, 8o.pspnc, 22ti.pspnc. Please see the attached input files. I think these should be a good start for you. Here are the brief details of what was done.

For ground state and qpscgw calculations, band_step1_2.in,
1. In order to save time, the calculations were performed at smaller ecut of 12, nbandkss of 50, nbands for G and W of 50. For the same reason, the qpscgw was also shorten to only two iterations (ndtset was set to 6). So please be aware that the results are poorly converged.
2. In dataset6, "npweps3 27" was replaced with "npwsigx4 27" because it is npwsigx that is needed in the sigma calculation. This seem to be the only mistake in your original input file "band_step1&2.in".
3. "bdgw" was set to "1 30" for all kptgw. No obvious reason for this, but I just to make sure that all the bands to be used in the wannier interpolation are gw corrected.
4. The generation of alchemical pseudoatoms was disable. (I don't have experience with this feature).
5. The gwpara 2 was introduced.

For wannier90 run for band interpolation, band_step3.in, a number of major mistakes were corrected.
1. The following lines were added to the input file "nband_step3.in" because they are still needed. The missing of these lines was probably the cause of errors you encountered.

----------------------------------------------------------------------------------------------
#Common to all model GW calculations
rhoqpmix 1.0
nkptgw 40
kptgw
0.00000000E+00 0.00000000E+00 0.00000000E+00
1.66666667E-01 0.00000000E+00 0.00000000E+00
.....
bdgw
1 30
1 30
.......
-------------------------------------------------------------------------------------------------
2. In the wannier90 input files *o_w90.win" the initial projections and the unit cell primitive vectors were incorrectly given in you original file. These are properly assigned as follows. Note that the initial projections may not be the best choices, though. The Cartesian primitive vectors were grasped from dataset1 of the ground state calculation, and multiplied by 0.529 A/Bohr to convert to Angstrom.
----------------------------------------------------------------------
begin projections
!site:ang_mtm:zaxis:xaxis:radial:zona:box-size
random
O:p
end projections
----------------------------------------------------------------------
begin unit_cell_cart
3.87123787 0.00000000 0.00000000
0.00000000 3.87123787 0.00000000
0.00000000 0.00000000 4.26239634
