Hello Raff,

I built the etsf.eu packages on XP sp3 (i.e. 32 bit binaries) but it would be quite surprising that a service pack level changes the behavior. It has been tested successfully with Windows 7 too (32 or 64 bit version). I've provided separate distributions for Intel or AMD processors - it's maybe a bit confusing; anyway be sure to grab the right one otherwise the binaries will immediately fail. Another possibility - if you run Abinit in parallel with the provided mpiexec launcher - is that Windows firewall asks to authorize the application to communicate - say yes, nothing fishy here: MPI uses sockets between threads, nothing goes on Internet.

Kind regards,

Alain
