In your abinit version, only the tests in paral and mpiio can be executed in parallel.
Moreover one can only use a number of MPI processes per test that must be in the list [1,2,4,10]

Use

runtests.py paral mpio -n 2

to run all the parallel tests with two MPI nodes.
If you have more than two (physical) cores on your machines, you can speedup the execution by running the tests in parallel
with python threads

runtests.py paral mpiio -n 2 -j 2

will use 2 (high-level) python threads to run the tests, each test uses 2 MPI processes so in total you will be using 4 CPUs.

The option
-n #num
is not supported in the other test suites, but you can use -j num_threads to speedup the execution of the sequential tests.
